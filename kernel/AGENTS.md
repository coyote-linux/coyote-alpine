# kernel

## OVERVIEW

Kernel build inputs, extracted kernel trees, and generated kernel/module archives. Treat this directory mostly as a boundary between maintained build inputs and generated/vendor content.

## WHERE TO LOOK

| Task | Location | Notes |
|------|----------|-------|
| Kernel build wrapper | `build-kernel.sh` | canonical custom-kernel entry |
| Maintained config inputs | `configs/`, `alpine-linux-lts.config` | regenerate carefully |
| Upstream source archives | `linux-*.tar.xz` | vendor inputs |
| Extracted sources | `linux-*/` | derived; avoid hand edits |
| Generated outputs | `output/` | module/kernel archives |

## CONVENTIONS

- `build/Makefile` calls `kernel/build-kernel.sh` when `CONFIG_KERNEL_TYPE=custom`.
- Built kernel image/modules are consumed by initramfs and installer build steps.
- Exposed config files can be regenerated by kernel tooling; preserve generated-file markers.

## ANTI-PATTERNS

- Do not patch extracted `linux-*` trees as the way to change product behavior.
- Do not edit `kernel/output/` artifacts.
- Do not remove tarballs/configs without checking build cache and Makefile fallback behavior.
- Do not edit `configs/coyote-x86_64.defconfig` casually; it is marked auto-generated.

## COMMANDS

```bash
./build-kernel.sh
cd ../build && make kernel
```
